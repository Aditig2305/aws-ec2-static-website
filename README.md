# Mini Project: Web Hosting on AWS EC2 with EBS Volume Backup

## Project Overview
This mini project demonstrates hosting a static website on an AWS EC2 instance with data persistence using an EBS volume backup.  
The project focuses on basic AWS cloud services and real-time deployment.

## Objective
- To host a static website on AWS EC2
- To ensure data reliability using EBS volume backup
- To understand basic EC2, EBS, and security group configuration

## AWS Services Used
- Amazon EC2
- Amazon EBS
- Security Groups
- Amazon Linux
- Apache Web Server

## Project Architecture
- EC2 instance launched in AWS
- EBS volume attached to EC2
- Apache installed to host website
- Website data stored on EBS volume
- EBS snapshot taken as backup

## Implementation Steps
1. Launched an Amazon EC2 instance
2. Configured security group to allow HTTP and SSH access
3. Attached an EBS volume to the EC2 instance
4. Installed Apache web server
5. Deployed static website files
6. Created EBS snapshot for backup and data recovery

## Key Features
- Static website hosting on EC2
- Persistent storage using EBS volume
- Backup using EBS snapshot
- Basic cloud security configuration

## Outcome
Successfully hosted a static website on AWS EC2 with reliable storage and backup using EBS volume.

## Learning Outcomes
- Hands-on experience with EC2 and EBS
- Understanding of cloud storage and backup concepts
- Basic AWS deployment workflow
- Real-world cloud project experience
